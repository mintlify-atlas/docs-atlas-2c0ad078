# GEMS LMS API

GEMS LMS API is a reactive microservices platform built on Spring Boot 3 and Spring WebFlux. It powers a Learning Management System (LMS) with distinct services for authentication, education content, and administration — all unified behind a single API Gateway.

## Key features

<CardGroup cols={2}>
  <Card title="Reactive architecture" icon="bolt">
    Built on Spring WebFlux and R2DBC for fully non-blocking, reactive I/O throughout the stack
  </Card>
  <Card title="API Gateway" icon="shield">
    Single entry point on port 8080 routes requests to the appropriate microservice
  </Card>
  <Card title="Clean architecture" icon="layer-group">
    Each service follows hexagonal architecture with domain, application, and infrastructure layers
  </Card>
  <Card title="Per-service databases" icon="database">
    Each microservice owns its PostgreSQL database, enforcing data isolation and independent scaling
  </Card>
  <Card title="Health monitoring" icon="heart-pulse">
    Spring Actuator exposes `/actuator/health`, `/actuator/info`, and `/actuator/metrics` on every service
  </Card>
  <Card title="Docker native" icon="docker">
    Full Docker Compose orchestration for local development with healthy service dependencies
  </Card>

## Services overview

The platform is composed of four Spring Boot applications:

| Service | Port | Responsibility |
|---|---|---|
| `api-gateway` | 8080 | Unified entry point, request routing |
| `ms-auth` | 8082 | User authentication and account management |
| `ms-education` | 8081 | Course and education content management |
| `ms-admin` | 8083 | Platform administration and admin user management |
